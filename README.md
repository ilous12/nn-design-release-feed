# design for air 시작 가이드

design for air는 사내 디자인 업무의 시작점을 로컬 작업 환경으로 옮기는 데스크톱 앱입니다. 요구사항을 넣고 화면을 생성하는 데서 끝나지 않고, 프로젝트 폴더, 디자인 시스템, 사내 템플릿, 검토 기준을 한 흐름 안에서 다룹니다.

## 설치

[최신 릴리스](https://github.com/ilous12/nn-design-release-feed/releases/latest)에서 운영체제에 맞는 설치 파일을 받아 실행합니다.

## 운영 방향

- **플랫폼에 묶이지 않는 실행 구조**: Claude Design 하나에 업무량, 모델, 저장 위치를 맡기지 않습니다.
- **로컬 CLI 우선**: Codex, Claude Code, Gemini CLI, 로컬 모델을 팀 정책에 맞춰 선택합니다.
- **비용 선택권**: API만 고집하지 않고 정액제 CLI, BYOK, 사내 키, 로컬 모델을 함께 고려합니다.
- **데이터 통제**: 디자인 파일, 프롬프트, 결과물, 프로젝트 컨텍스트를 로컬 또는 사내 승인 경로에서 관리합니다.
- **회사 기준의 재사용**: 사내 서비스 UI, 브랜드 톤, 컴포넌트 규칙, 개발팀 산출물 형식에 맞는 결과물을 남깁니다.

## 가이드

- **열기**: 프로젝트 폴더와 디자인 시스템을 먼저 연결해 작업 맥락을 고정합니다.
- **작성**: 질문, 계획, 디자인 모드로 요구사항을 화면 구조와 실행 계획으로 바꿉니다.
- **리믹스**: 기존 레퍼런스를 회사 톤, 업무 플로우, 개발 전달 기준에 맞게 다시 구성합니다.
- **설정**: Codex, Claude Code, Gemini CLI, 로컬 모델, API 키를 보안 정책과 비용 기준에 맞춥니다.
- **검토**: 결과를 바로 배포 대상으로 보지 않고, 수정과 재사용이 가능한 내부 산출물로 정리합니다.

## 배포 구조

이 저장소에는 앱이 참조하는 작은 메타데이터만 보관합니다. 용량이 큰 설치 파일, 런처 payload, 체크섬 파일은 각 버전의 GitHub Release asset으로 업로드됩니다.
