중요한 건 모델보다
플랫폼 비종속과 데이터 통제입니다

nn.design은 Claude Design에 업무를 종속시키지 않고, 회사가 허용한 CLI와 모델을 로컬 중심으로 묶어 운영합니다.

01

플랫폼 비종속 운영

특정 플랫폼의 사용량, 모델, 저장 방식에 묶이지 않고 회사 기준으로 실행 경로를 선택합니다.

02

로컬 CLI 우선 연결

Codex, Claude Code, Gemini CLI, 로컬 모델을 작업 PC 또는 사내 승인 환경에서 먼저 사용합니다.

03

전체 작업 로컬 통제

디자인 파일, 프롬프트, 산출물, 프로젝트 컨텍스트를 로컬 또는 사내 승인 경로에서 관리합니다.

04

회사 맞춤 디자인/업무 시스템

Open.design의 범용 시스템을 그대로 쓰지 않고 사내 서비스 UI, 브랜드 톤, 컴포넌트 규칙, 개발팀 산출물 형식에 맞게 정제합니다.

05

결과물 재사용 기준

생성 결과는 검토, 수정, 재사용할 수 있는 형태로 남기고 팀의 품질 기준에 맞춰 관리합니다.

₩

정액제와 비용 분산

API 방식만 고집하지 않고 정액제 CLI, BYOK, 사내 키, 로컬 모델을 조합해 비용과 사용량 한도를 분산합니다.

작업은 로컬에서 시작하고
필요한 도구만 선택합니다

초안 생성, 리믹스, 코드 변환, 리뷰를 같은 프로젝트 컨텍스트 안에서 이어가되 API, 정액제 CLI, 로컬 모델 중 팀 정책에 맞는 방식을 선택합니다.

열기

프로젝트 컨텍스트 연결

작업 폴더, 사내 템플릿, 디자인 시스템 기준을 먼저 열어 AI가 회사 맥락 안에서 답하게 만듭니다.

선택

목적에 맞는 CLI 지정

Codex, Claude Code, Gemini CLI, 로컬 모델 중 작업 성격과 보안 기준에 맞는 실행 경로를 선택합니다.

리믹스

결과를 업무 형식으로 재구성

관리자 화면, 모바일 우선, AIR 톤, 개발 전달용 스펙처럼 같은 요구를 여러 산출물 형태로 다시 만듭니다.

검토

결과를 내부 기준으로 확인

생성물은 그대로 끝내지 않고 검토, 수정, 재사용 가능한 형태로 정리합니다.

설치는 가볍게,
실행은 로컬 기준으로

macOS와 Windows 모두 같은 활용 흐름을 기준으로 안내하고, 설치 후에는 팀에서 허용한 CLI와 키 관리 방식을 설정합니다.

설치: 최신 릴리스에서 현재 운영체제에 맞는 설치 파일을 받아 실행하세요.
