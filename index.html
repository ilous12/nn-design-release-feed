디자인 업무의 시작점을
회사 안에 둡니다

design for air는 요구사항, 레퍼런스, 디자인 시스템, 로컬 CLI 실행을 한 작업 흐름으로 묶습니다. 어떤 모델을 쓸지보다 중요한 것은 업무 맥락과 데이터를 누가 통제하느냐입니다.

OS에 맞는 설치 파일 받기 활용 방식 보기
설치는 운영체제에 맞는 파일을 받으면 끝납니다. 실제 운영은 팀에서 허용한 CLI, 키, 로컬 모델 기준으로 맞춥니다.

외부 플랫폼을 쓰더라도
업무의 기준은 내부에 남겨야 합니다

design for air는 Claude Design 하나에 사용량, 모델, 저장 위치를 맡기지 않습니다. 회사가 선택한 실행 경로 안에서 디자인 작업을 이어가도록 돕습니다.

01

플랫폼 비종속

특정 서비스의 한도와 정책에 업무 일정을 맞추지 않습니다. 실행 경로는 회사 기준으로 선택합니다.

02

로컬 CLI 우선

Codex, Claude Code, Gemini CLI, 로컬 모델을 작업 PC 또는 사내 승인 환경에서 먼저 연결합니다.

03

데이터 통제

디자인 파일, 프롬프트, 산출물, 프로젝트 컨텍스트가 어디에 남는지 명확히 관리합니다.

04

회사 맞춤 시스템

범용 디자인 시스템을 그대로 쓰지 않고 서비스 UI, 브랜드 톤, 컴포넌트 규칙에 맞게 정제합니다.

05

재사용 가능한 결과물

한 번 만든 화면을 대화 기록으로 흘려보내지 않고, 검토와 수정이 가능한 산출물로 남깁니다.

₩

비용 선택권

API 방식만 고집하지 않고 정액제 CLI, BYOK, 사내 키, 로컬 모델을 함께 고려합니다.

하나의 답을 받는 도구가 아니라
작업을 이어가는 자리입니다

초안 작성, 리믹스, 코드 변환, 검토를 같은 프로젝트 맥락에서 이어갑니다. 실행은 API, 정액제 CLI, 로컬 모델 중 팀 정책에 맞게 선택합니다.

열기

맥락을 먼저 고정

작업 폴더, 사내 템플릿, 디자인 시스템을 먼저 열어 결과가 회사 맥락을 벗어나지 않게 합니다.

선택

도구는 목적별로 선택

Codex, Claude Code, Gemini CLI, 로컬 모델 중 작업 성격과 보안 기준에 맞게 고릅니다.

리믹스

업무 형식으로 리믹스

관리자 화면, 모바일 우선, 서비스 톤, 개발 전달용 스펙처럼 같은 요구를 다른 산출물로 바꿉니다.

검토

검토 후 남기는 방식

생성물은 그대로 끝내지 않고 팀 기준에 맞게 다듬어 다음 작업에 다시 씁니다.

설치는 짧고,
운영 기준은 분명하게

macOS와 Windows 모두 같은 흐름으로 안내합니다. 설치 후에는 팀에서 허용한 CLI와 키 관리 방식을 맞추면 됩니다.

최신 릴리스에서 현재 운영체제에 맞는 설치 파일을 받아 실행하세요.

도구는 바뀌어도, 작업 기준은 유지됩니다

Codex, Claude Code, Gemini CLI, 로컬 모델을 필요한 만큼 연결하고 결과물은 회사 기준 안에서 관리합니다.
