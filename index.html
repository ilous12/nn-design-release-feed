Start in three steps

초심자도 그대로 따라갈 수 있는 설치 흐름입니다.

1

파일 선택

macOS 사용자는 최신 릴리스에서 .dmg 파일을 받습니다. payload는 업데이트 과정에서 앱이 자동으로 사용합니다.

2

앱 설치

DMG를 열고 Design For AIR를 Applications 폴더로 이동합니다. 기존 버전은 종료 후 교체합니다.

3

첫 실행

보안 확인이 나오면 배포 출처를 확인한 뒤 실행하고, 설정 화면에서 언어와 에이전트 상태를 점검합니다.

Guide for open and remix

설치 후에는 열기, 리믹스, 설정 순서로 작업 기준을 잡습니다.

DMG 기준 배포

사용자 설치는 DMG를 기준으로 안내합니다.

Payload 업데이트

런처 payload는 자동 업데이트용 artifact로 함께 배포합니다.

버전 검증

앱 버전이 릴리스 태그보다 낮을 때 업데이트가 감지됩니다.

같은 버전의 앱과 feed를 비교하면 최신 상태로 표시됩니다.

열기

작업할 프로젝트 폴더나 사내 템플릿 폴더를 엽니다.

리믹스

관리자 화면, 모바일 우선, 사내 서비스 톤처럼 목적을 바꿔 다시 요청합니다.

설정

허용된 에이전트, API 키, 언어, 디자인 시스템을 조직 정책에 맞춥니다.

업데이트와 검증

앱은 아래 메타데이터를 읽어 최신 버전과 플랫폼별 artifact를 확인합니다. GitHub Release asset에는 DMG, payload, SHA-256 체크섬이 함께 올라갑니다.
